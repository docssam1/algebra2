# PATCH_PLAN_ALGEBRA2_FINALIZATION

## 목표

Algebra 2 Placement Dashboard를 단순 채점 화면이 아니라 아래 흐름의 완성형 수업/상담 시스템으로 정리한다.

```text
진단 시험지
→ OMR 입력
→ 학부모 분석 리포트
→ 약점 기반 수업용 개념
→ Khan Academy 영상 연결
→ 맞춤 워크북
→ 워크북 해설지
→ 최종 Intro 화면
→ Admin / Teacher 관리 화면
```

## 운영 원칙

- 실제 코드 수정은 가능하면 VM + Telegram + git commit/push 흐름으로 진행한다.
- 대형 `index.html` 전체 덮어쓰기는 금지한다.
- 패치 전 반드시 `/run algebra2_backup`을 실행한다.
- 패치 후 반드시 `/run algebra2_diff`, `/run algebra2_test`를 실행한다.
- 기능은 작은 단위로 나누어 반영한다.

## 확정 수정 목록

### 1. 한영 토글 완전 정리

영어 모드에서는 모든 메뉴, 버튼, 리포트, 경고문, 워크북, 해설지에 한글이 없어야 한다.

대상:

- 상단 메뉴
- 탭 메뉴
- 버튼
- OMR 입력보드
- 분석 리포트
- 로드맵
- 워크북
- 해설지
- 안내/알림 문구

### 2. 난이도 표현 수정

- `저난도` → `저난이도 문제`
- `저난도 문항` → `저난이도 문제`

### 3. 1~40번 정오답 + 난이도 표 추가

학부모 분석지에 1번~40번의 정오답과 난이도를 표시한다.

권장 구조:

```text
1  2  3  ... 20
O  X  -  ... O
B  C  A  ... B

21 22 23 ... 40
O  X  -  ... O
B  C  A  ... A
```

표기:

- O = 정답
- X = 오답
- - = 미채점
- B = 저난이도
- C = 중난이도
- A = 고난이도

### 4. 고난이도 정답 + 저난이도 오답 분석

같은 유형 안에서 고난이도 문제는 맞히고 저난이도 문제를 틀린 경우, 개념 부족보다 아래 가능성을 우선 진단한다.

- 계산 실수
- 조건 확인 누락
- 부호 실수
- 답안 전사 실수
- 성급한 풀이

### 5. 깨진 수식 제거

동적 리포트 안에서 MathJax가 렌더링되지 않아 깨지는 수식은 일반 텍스트로 변경한다.

예:

```text
(a-b)(a+b)=a²-b²
```

### 6. 세부 영역 그래프 추가

학부모 분석지에 세부 영역별 그래프 2종을 추가한다.

1. 원그래프
   - 세부 영역별 오답 비중
2. 비교 그래프
   - 세부 영역별 성취도 비교

대상 영역:

```text
Linear & Coord
Quadratics & Complex
Polynomials
Rational & Radical
Exponents & Logs
Trigonometry
Discrete Math & Stats
Systems & Matrices
```

### 7. OMR 입력보드 가로 2줄형 변경

현재 세로형 OMR 입력보드를 가로 2줄형으로 변경한다.

권장 구조:

```text
Q1  Q2  Q3 ... Q20
B   C   A  ... B
O/X O/X O/X ... O/X

Q21 Q22 Q23 ... Q40
B   C   A  ... A
O/X O/X O/X ... O/X
```

### 8. AI 서버 상태 표시 단순화

상단에는 아래 형태만 표시한다.

```text
AI 서버: ● 연결 중
AI 서버: ● 정상 연결
AI 서버: ● 연결 실패
```

숨김 대상:

- Gemini 엔진
- Google Cloud Gemini Proxy
- 서버 확인 버튼

### 9. 진단 문제지 출력

학생용 진단 문제지를 출력할 수 있어야 한다.

포함:

- 1~40번 문제
- Part 1 / Part 2 구분
- 풀이 공간
- 답란
- OMR 입력용 번호 구조

### 10. 진단 문제지 답지/해설지 출력

진단 문제지에 대한 답지와 해설지를 출력할 수 있어야 한다.

포함:

- 문제 번호
- 정답
- 핵심 풀이
- 관련 단원
- 난이도

### 11. 학부모 분석지 고도화

학부모 분석지에는 아래가 포함되어야 한다.

- 총점
- 정답률
- 세부 영역 분석
- 난이도별 분석
- 1~40 정오답/난이도표
- 계산 실수 가능성 분석
- 선수개념 결손 분석
- 원그래프
- 비교 그래프

### 12. 워크북 실제 문항 출력

워크북은 회차표만이 아니라 실제 문항을 포함해야 한다.

포함:

- 회차명
- 약점 영역
- 수업 목표
- 실제 문제
- 풀이 공간
- 답란
- 숙제 문제

### 13. 워크북 해설지 출력

워크북과 같은 문항 순서로 해설지가 출력되어야 한다.

포함:

- 정답
- 핵심 풀이
- 오답 원인
- 다음 수업 연결 포인트

### 14. Khan Academy 연결 기반 수업용 개념 페이지

워크북의 개념 페이지는 칸아카데미 내용을 그대로 복사하지 않는다.

구조:

```text
Khan Academy = 영상 링크 / 개념 흐름
G-FIELD = DOCSSAM식 수업 포인트 / 오답 원인 / 적용법 / 대표 예제 / 숙제 구성
```

각 회차 개념 페이지 포함:

- 오늘의 핵심 개념
- Khan Academy 영상 제목과 링크
- 시청 순서
- 시청 전 질문
- 시청 후 확인 문제
- DOCSSAM 수업 포인트
- 대표 예제
- 확인 문제

### 15. OCR 보조 입력기는 나중 단계

시험지 사진/PDF OCR 기능은 본체 완성 후 2단계로 진행한다.

역할:

- 사진/PDF 업로드
- OCR 텍스트 추출
- 문제 번호별 자동 분리
- 원장님 검수 후 확정
- 확정 데이터만 OMR/분석 리포트에 반영

자동 채점 핵심 엔진으로 바로 사용하지 않는다.

### 16. Intro 화면 추가

본체 기능 완성 후 Intro 화면을 추가한다.

포함:

- G-FIELD 브랜드 화면
- 학생 이름 입력
- 진단 시작
- 기존 학생 불러오기
- 학부모 안내
- 관리자/수업용 진입 버튼

### 17. Admin / Teacher 관리 화면 추가

마지막 확장 단계에서 Admin 화면을 추가한다.

포함:

- 학생 목록 관리
- 진단 기록 불러오기
- OMR 결과 저장/수정
- 리포트 재생성
- 워크북/해설지 재생성
- Khan Academy 링크 매칭 관리
- 문제 난이도/영역 수정
- 학부모 공유용 링크 생성

## 패치 순서

### Phase 1: 현재 화면 안정화

1. 깨진 수식 제거
2. 저난이도 문구 수정
3. AI 서버 상태 표시 최종 정리
4. OMR 입력보드 가로 2줄형 변경
5. 한영 토글 1차 정리

### Phase 2: 분석 리포트 고도화

1. 1~40번 정오답/난이도표 추가
2. 계산 실수 가능성 분석 추가
3. 원그래프 추가
4. 비교 그래프 추가
5. 한영 토글 2차 정리

### Phase 3: 출력물 완성

1. 진단 문제지 출력
2. 진단 답지/해설지 출력
3. 학부모 분석지 HTML/PDF 출력 정리
4. 워크북 실제 문항 출력
5. 워크북 해설지 출력

### Phase 4: 수업용 개념/영상 연결

1. Khan Academy 매칭 데이터 연결
2. 회차별 수업용 개념 페이지 생성
3. 대표 예제/확인 문제/숙제 구성

### Phase 5: Intro/Admin/OCR

1. Intro 화면
2. Admin 화면
3. OCR 보조 입력기

```text
/run algebra2_backup
→ 깨진 수식 제거 + 저난이도 문구 수정
→ /run algebra2_diff
→ /run algebra2_test
```
